# Serenotes

A personal note-taking app built with vanilla JavaScript, packaged as an Android app via Capacitor. Clean, minimal, and fully offline-capable using LocalStorage and IndexedDB — no backend required.

---

## Features

- **Create, edit, and delete notes** with title, content, category, and date
- **Category management** — custom categories and subcategories with icons and colors
- **Reminders** — set datetime-based reminders per note
- **Calendar view** — browse notes by date
- **Search** — full-text search across all notes
- **Media attachments** — attach images, audio, and video via IndexedDB
- **Pin notes** — star important notes to the top
- **Backup & restore** — export all notes as JSON
- **PWA-ready** — manifest + service worker included
- **Android app** — wrapped with Capacitor for native Android

## Getting Started

### Run on browser (Live Server)

1. Open the `apps/` folder with **VS Code Live Server**
2. Open `http://127.0.0.1:5500/welcome.html`

> Make sure Live Server is pointed at the `apps/` folder, not the root.

### Build for Android

**Prerequisites:**
- Node.js v18+
- Android Studio
- JDK 17+

**Steps:**

```bash
# Install dependencies
npm install

# Sync web assets to Android project
npx cap sync android

# Open in Android Studio
npx cap open android
```

Then in Android Studio, click **Run ▶** to deploy to a device or emulator.

**Get the APK directly:**
```
android/app/build/outputs/apk/debug/app-debug.apk
```

## Data Storage

All data is stored **locally on device** — no server, no account needed.

| Data | Storage |
|---|---|
| Notes, categories, reminders | `localStorage` |
| Media attachments (images, audio, video) | `IndexedDB` |
| User name | `localStorage` (`serenotes_user`) |

---

## Key Services

| Service | Responsibility |
|---|---|
| `noteService.js` | CRUD notes, pin, archive, search |
| `categoryService.js` | CRUD categories & subcategories |
| `reminderService.js` | Create, complete, and query reminders |
| `mediaService.js` | Upload, retrieve, and delete media via IndexedDB |
| `searchService.js` | Full-text search across notes |

---

## Development Notes

- After any change to `apps/`, run `npx cap sync android` before testing on device
- Service Worker (`sw.js`) is inactive inside Capacitor — offline is handled natively by LocalStorage/IndexedDB
- `manifest.json` is not used by Capacitor — app metadata is configured via Android Studio
